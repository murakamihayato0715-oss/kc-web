# 開発・設計における絶対遵守規約 (Architectural Principles)

## 1. AIと確定プログラムロジックの役割分担に関する絶対原則
- **AIは「人間（提督）と決定論的プログラムロジックをつなぐ自然言語インターフェース（NLU）」に過ぎない。**
- 艦これのように提督一人ひとりの所有艦娘、所持装備、改修値（★+N）、補強増設の有無、パラメータ・フィットボーナスが複雑に異なるゲームにおいて、生成AI（LLM）の推測やテキスト生成能力にステータス計算やアセンブル決定を任せることは不可能です。AIが正確な情報を得て自律判断できると過信してはいけません。
- **すべてのステータス計算、射程判定、スロット制限チェック、所持在庫照合は、必ず制空権シミュレーター（`kc-web`）本体の実体プログラムコード（`ShipValidation`, `Item`, `Ship`, `equipmentSolver`）を直接実行して100%決定論的に算出・確定させなければならない。**
- AIへ渡すプロンプトには、プログラムコード側で事前検証・確定させた「100%正確な確定データ」のみを注入し、AIにはその結果の解説や人間への応答生成のみを担当させなければならない。

## 2. 応答テキストおよび対話スタイルの厳格ルール
- **挨拶・前置き・メタ解説の完全禁止**:
  チャット応答およびAI解説テキストにおいて、「提督、」「承知いたしました」「～についてご説明します」などの挨拶・前置き、およびシステム内部のルールやプロンプト（「確定装備リスト」「厳格ルール」等）への言及を固く禁止する。
- **結論直球出力**:
  余計な挨拶や言い訳、メタ解説を一切排除し、1文字目から要点・編成結果・解説のみを簡潔に出力すること。
